# Z7 PDF 工作台

一个本地部署的 PDF 在线编辑工具，默认通过 `docker compose` 运行在 `39010` 端口，支持会员空间、在线保存、文件分享、回收站、套餐订阅和后台管理。

## 功能特性

### 可视化编辑台
- 支持点击或拖拽 PDF 到工作台空状态区域直接开始编辑
- 多文件导入、拖拽排序、单页预览
- 插入空白页、删除、旋转、裁剪、标注
- 压缩、加密、页码、水印、页眉页脚、PDF/A、OCR
- 单文件导出或按每 N 页拆分导出

### 会员工作区
- 密码登录 / 邮箱验证码登录
- 未注册邮箱验证后自动注册
- 个人文件空间、在线保存编辑结果
- 历史文件再次打开继续编辑
- 多级目录树、批量打开 PDF
- 文件重命名、移入回收站、恢复和彻底删除

### 文件分享
- 公开链接、密码访问、登录访问三种模式
- 设置到期时间和下载次数限制

### 套餐与订阅
- 会员提交订阅订单
- 管理员审核后自动激活订阅并更新权益

### 后台管理
- 查看会员、文件、分享和订单概览
- 配置站点名称、默认套餐、注册开关、空间配额
- 配置支付提示、SMTP 发信参数
- 直接调整会员角色和套餐

### PDF 处理功能
| 功能 | 说明 |
|------|------|
| 合并 | 合并多个 PDF 文件 |
| 压缩 | 低 / 中 / 高三档压缩 |
| 页面整理 | 拖拽排序、插入空白页、删除页面 |
| 页面旋转 | 全部页、奇数页、偶数页、指定页码 |
| 页面尺寸 | A3 / A4 / A5 / Letter / Legal、方向、边距、缩放 |
| 拆分 | 按页码范围或每 N 页拆分，输出 ZIP |
| 水印与页码 | 指定页面、位置、透明度、颜色、对齐方式 |
| 加密与解密 | 密码加密和去密码 |
| OCR 文字识别 | 基于 Tesseract 与 OCRmyPDF，支持图片 PDF 转双层可搜索 PDF |

## Docker 架构

项目现在采用标准的 Docker 分层方式：
- `Dockerfile` 使用 `base / deps / runtime` 多阶段结构，区分系统依赖、Node 依赖和最终运行层。
- `.dockerignore` 会过滤 `data/`、`.git/`、本地工具历史和日志，避免把无关文件打进镜像上下文。
- `docker-compose.yml` 负责生产运行。
- `docker-compose.dev.yml` 只覆盖开发态命令与源码挂载。
- 容器默认以非 root 的 `node` 用户运行，并带 `/health` 健康检查。

详细说明见 [docs/docker.md](docs/docker.md)。

## 快速开始

### Docker（推荐）

#### 构建标准镜像
```bash
docker build -t z7pdf:latest .
```

#### 直接运行镜像
```bash
docker run -d \
  --name z7pdf \
  -p 39010:39010 \
  -v "$(pwd)/data:/app/data" \
  z7pdf:latest
```

#### 生产模式
```bash
docker compose up -d --build
```

#### 开发模式（修改代码后自动生效）
```bash
docker compose -f docker-compose.yml -f docker-compose.dev.yml up -d --build
```

开发模式特点：
- ✅ 修改 `server.js`、`db.js` 自动重启
- ✅ 修改 `public/`、`routes/`、`services/`、`middleware/`、`utils/` 立即生效
- ✅ 无需频繁重建镜像

数据持久化：`./data` 挂载到容器 `/app/data`，包含 SQLite 数据库和用户文件。

启动后访问：
- 主页：http://127.0.0.1:39010
- 管理后台：http://127.0.0.1:39010/admin.html
- 分享页面：http://127.0.0.1:39010/share.html?token=...

停止服务：

```bash
docker compose down
```

### 本地 Node.js 运行

```bash
npm install
npm start
```

> 注意：本地运行需要安装 `ghostscript` 和 `qpdf` 以支持压缩和加密功能。
>
> 如需启用 OCR，还需要安装 `ocrmypdf`、`unpaper`、`tesseract-ocr`、`tesseract-ocr-eng`、`tesseract-ocr-chi-sim`。

## 技术栈

| 组件 | 用途 |
|------|------|
| Node.js 22 + Express | Web 服务 |
| SQLite (node:sqlite) | 用户、会话、文件、分享、订单等数据存储 |
| pdf-lib | 页面复制、重排、旋转、尺寸调整、水印、页码 |

| Ghostscript | PDF 压缩 |
| qpdf | PDF 加密与解密 |
| OCRmyPDF & Tesseract | 文档 OCR 文字识别引擎 |
| JSZip | 拆分结果打包 |
| multer | 文件上传处理 |
| nodemailer | 邮件验证码发送 |

## 项目结构

```text
z7pdf/
├── server.js           # 主服务入口（仅作路由挂载与初始化）
├── db.js               # 底层 SQLite 数据库 DAO 与表结构定义
├── routes/             # 路由控制层（按业务领域：auth, workspace, share 等）
├── services/           # 核心服务层（业务逻辑与通用能力封装）
├── middleware/         # Express 拦截与校验中间件
├── utils/              # 通用工具函数库
├── package.json        # 依赖配置
├── Dockerfile          # Docker 镜像构建
├── docker-compose.yml  # Docker Compose 配置
├── docker-compose.dev.yml # 开发态 Docker Compose 覆盖配置
├── docs/
│   └── docker.md       # Docker 架构与镜像构建说明
├── public/             # 前端静态文件
│   ├── index.html      # 主页
│   ├── admin.html      # 管理后台
│   ├── share.html      # 分享页面
│   ├── editor.js       # 编辑器逻辑
│   ├── workspace.js    # 工作区与账户逻辑
│   ├── admin.js        # 后台逻辑
│   └── styles.css      # 样式
└── data/               # 数据目录（持久化）
    ├── app.db          # SQLite 数据库文件
    └── storage/        # 用户文件存储
```

### 工作台使用说明

#### 顶部标签

| 标签 | 功能说明 |
|------|----------|

#### 文件操作

- **上传**：点击左侧上传按钮，或直接点击 / 拖拽 PDF 到右侧编辑区空状态区域
- **打开编辑**：点击文件名或"继续编辑"按钮
- **创建分享**：点击文件右侧"创建分享"按钮生成分享链接
- **移到回收站**：将不需要的文件移入回收站（可恢复）
- **彻底删除**：从回收站删除文件（不可恢复）
