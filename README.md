# Z7 PDF 工作台

一个本地部署的 PDF 在线编辑工具，默认通过 `docker compose` 运行在 `39010` 端口，支持会员空间、在线保存、文件分享、回收站、兑换码会员和后台管理。

## 功能特性

### 可视化编辑台
- 支持点击或拖拽 PDF 到工作台空状态区域直接开始编辑
- 多文件导入、拖拽排序、单页预览
- 插入空白页、删除、旋转、裁剪、标注
- 压缩、加密、页码、水印、页眉页脚、PDF/A、OCR
- 单文件导出或按每 N 页拆分导出

### 会员工作区
- 密码登录 / 邮箱验证码登录
- 未注册邮箱验证后自动注册
- 个人文件空间、在线保存编辑结果
- 历史文件再次打开继续编辑
- 多级目录树、批量打开 PDF
- 文件重命名、移入回收站、恢复和彻底删除

### 文件分享
- 公开链接、密码访问、登录访问三种模式
- 设置到期时间和下载次数限制

### 会员开通与有效期
- 管理员后台生成会员兑换码
- 用户通过兑换码开通或续期会员
- 管理员可直接调整套餐和会员有效期

### 后台管理
- 查看会员、文件、分享和有效会员概览
- 配置站点名称、默认套餐、注册开关、空间配额
- 配置 SMTP 发信参数
- 直接调整会员角色、套餐和有效期

### 隐私与安全
- 自部署运行，文件处理和会员空间默认都保存在当前服务器的 `data/` 目录
- 前端 PDF 依赖由本服务本地提供，不依赖第三方 CDN
- `docker-compose.yml` 默认不写入管理员密码、SMTP 密码、Token 或 API Key
- 分享链接支持密码、到期时间和下载次数限制

### PDF 处理功能
| 功能 | 说明 |
|------|------|
| 合并 | 合并多个 PDF 文件 |
| 压缩 | 低 / 中 / 高三档压缩 |
| 页面整理 | 拖拽排序、插入空白页、删除页面 |
| 页面旋转 | 全部页、奇数页、偶数页、指定页码 |
| 页面尺寸 | A3 / A4 / A5 / Letter / Legal、方向、边距、缩放 |
| 拆分 | 按页码范围或每 N 页拆分，输出 ZIP |
| 水印与页码 | 指定页面、位置、透明度、颜色、对齐方式 |
| 加密与解密 | 密码加密和去密码 |
| OCR 文字识别 | 基于 Tesseract 与 OCRmyPDF，支持图片 PDF 转双层可搜索 PDF |

## Docker 架构

项目现在采用标准的 Docker 分层方式：
- `Dockerfile` 使用 `base / deps / runtime` 多阶段结构，区分系统依赖、Node 依赖和最终运行层。
- `.dockerignore` 会过滤 `data/`、`.git/`、本地工具历史和日志，避免把无关文件打进镜像上下文。
- `docker-compose.yml` 负责生产运行。
- 容器默认以非 root 的 `node` 用户运行，并带 `/health` 健康检查。
- 默认使用宿主机目录 `./data` 持久化 `/app/data`，便于直接备份和迁移。
- 容器启动时会自动尝试修正 `./data` 的目录权限，然后再降权到 `node` 用户运行。
- 当前 `docker-compose.yml` 默认只写入非敏感配置，并预留 `ADMIN_EMAIL`、`ADMIN_PASSWORD` 环境变量占位；真实密码应通过宿主机环境变量、未提交的 `.env` 或 Secret 管理系统注入。
- 前端 `pdf-lib` 与 `pdfjs-dist` 均从本服务 `/vendor/` 路径提供，避免浏览器访问第三方 CDN。

详细说明见 [DOCKER.md](DOCKER.md)。

## 快速开始

### Docker（推荐）

#### 构建标准镜像
```bash
docker build -t z7pdf:latest .
```

#### 直接运行镜像
```bash
docker run -d \
  --name z7pdf \
  -p 39010:39010 \
  -v "$(pwd)/data:/app/data" \
  z7pdf:latest
```

#### 生产模式
```bash
docker compose up -d --build
```

数据持久化：默认使用宿主机目录 `./data` 挂载到容器 `/app/data`，包含 SQLite 数据库和用户文件。

启动后访问：
- 主页：http://127.0.0.1:39010
- 管理后台：http://127.0.0.1:39010/admin.html
- 分享页面：http://127.0.0.1:39010/share.html?token=...

首次初始化说明：
- 如果数据库里还没有管理员账号，系统会自动创建一个管理员。
- 管理员邮箱默认是 `admin@z7pdf.local`，可用 `ADMIN_EMAIL` 覆盖。
- 如果未设置 `ADMIN_PASSWORD`，系统会在首次启动时生成随机密码、写入数据库，并打印到启动日志。
- 如果使用 `docker compose up -d --build` 后台启动，可通过 `docker compose logs --tail=50 z7pdf` 查看这组初始账号信息。

停止服务：

```bash
docker compose down
```

注意：`docker compose down` 不会删除 `./data` 目录。容器启动时会自动尝试修正该目录权限；如果你的宿主机策略不允许，手动执行 `sudo chown -R 1000:1000 data` 即可。

### 本地 Node.js 运行

```bash
npm install
npm start
```

> 注意：本地运行需要安装 `ghostscript` 和 `qpdf` 以支持压缩和加密功能。
>
> 如需启用 OCR，还需要安装 `ocrmypdf`、`unpaper`、`tesseract-ocr`、`tesseract-ocr-eng`、`tesseract-ocr-chi-sim`。

## 管理员初始化

首次启动且数据库中不存在该邮箱时：
- 默认邮箱为 `admin@z7pdf.local`
- 如果设置了 `ADMIN_PASSWORD`，会按该密码创建管理员
- 如果没有设置 `ADMIN_PASSWORD`，系统会自动生成随机密码并打印到启动日志

管理员账号只会在首次写库时自动创建一次。后续重启不会重复生成，也不会覆盖你已存在的管理员密码。

可通过环境变量覆盖：

```yaml
environment:
  ADMIN_EMAIL: "your-admin@example.com"
  ADMIN_PASSWORD: "your-secure-password"
```

使用 Docker Compose 时，不要把真实密码直接写进 `docker-compose.yml`；可以在命令前注入：

```bash
ADMIN_EMAIL=your-admin@example.com ADMIN_PASSWORD='your-secure-password' docker compose up -d --build
```

## 隐私与安全边界

### 数据保存位置

- SQLite 数据库、会员上传文件、在线保存结果、分享记录和兑换码记录保存在 `data/` 目录。
- 游客导出默认只触发浏览器下载，不会写入会员空间；但上传文件会在服务器临时目录中短暂处理，进程会在请求结束和定时任务中清理过期临时文件。
- 浏览器下载的文件、下载历史和本地缓存由用户设备和浏览器管理，处理敏感文件后请按本机策略清理。

### 外部网络请求

- 应用运行所需的前端 PDF 脚本由本服务本地提供，不再从第三方 CDN 加载。
- 邮箱验证码只在管理员配置 SMTP 后发送，邮件会通过你配置的 SMTP 服务商发出。
- OCR、压缩、加密和 PDF/A 等处理使用容器内安装的本地命令行工具，不需要把文件发送到外部 OCR 或 PDF 平台。

### 密钥与账号

- 不要把 `ADMIN_PASSWORD`、SMTP 密码、Token、私钥或证书提交到仓库。
- 生产环境建议通过宿主机环境变量、`.env`（不提交）或部署平台 Secret 注入敏感配置。
- 首次随机生成的管理员密码会出现在启动日志中，完成登录后建议立即修改，并控制日志访问权限。

### 公网部署建议

- 放到公网前建议使用 HTTPS 反向代理，并限制代理层和应用层的上传体积。
- 定期备份 `data/`，同时限制该目录的宿主机访问权限。
- 公开分享链接应视为访问凭证，敏感文件建议使用密码分享、到期时间和下载次数限制。
- 管理员可以访问用户、文件、套餐和分享记录，请只给可信人员管理员权限。

### 隐私与安全说明

| 问题 | 说明 |
|------|------|
| **文件会上传到第三方吗？** | 不会。PDF 处理在当前部署的服务器中完成，前端 PDF 脚本也由本服务本地提供。 |
| **游客文件会保存吗？** | 游客导出默认只下载到本机，不进入会员空间；上传处理过程中的临时文件会在请求结束和定时任务中清理。 |
| **会员文件保存在哪里？** | 会员主动上传或在线保存的文件保存在服务器 `data/` 目录，对应 SQLite 记录也在同一数据目录中。 |
| **分享链接如何保护？** | 敏感文件建议使用密码分享、到期时间、下载次数限制；不再需要时停用或删除分享。 |
| **管理员能看到什么？** | 管理员后台可管理用户、文件、分享、套餐和兑换码，因此管理员账号必须使用强密码并限制访问。 |
| **部署要注意什么？** | 生产环境建议启用 HTTPS、保护 `data/`、不要提交 Secret，并定期备份。 |
